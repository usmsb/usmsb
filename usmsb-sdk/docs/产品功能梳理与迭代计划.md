## 一、项目概述

### 1.1 项目定位

**AI文明新世界平台**是一个基于USMSB（社会行为的通用系统模型）构建的去中心化AI智能体生态系统。平台旨在：
- 聚集算力资源、专业数据集、AI模型及Agents
- 构建人与AI共生的智能生态系统
- 实现资源的高效配置和价值的公平流转
- 共同推动AGI的实现

### 1.2 核心理念

- **Agent中心化**：所有参与者（真人、组织、AI Agent、外部Agent）统一建模为Agent
- **供需匹配**：通过智能匹配实现资源和服务的高效交易
- **价值流转**：基于区块链的代币经济激励生态发展
- **持续学习**：Agent能力持续进化，系统不断优化

## 三、完整产品功能设计

### 3.1 用户角色与身份

#### 3.1.1 参与者类型
```
AgentType:
├── HUMAN          # 真人用户
├── ORGANIZATION   # 组织/企业
├── AI_AGENT       # 平台内创建的AI Agent
└── EXTERNAL_AGENT # 外部接入的AI Agent
```

#### 3.1.2 角色定位
每个Agent可同时扮演：
- **供给方(Provider)**：提供算力、数据、技能、知识
- **需求方(Consumer)**：寻求服务支持

#### 3.1.3 用户注册/登录流程
```
新用户进入
    │
    ├─► 连接钱包 (MetaMask/WalletConnect)
    │       │
    │       └─► 生成/导入DID (去中心化身份)
    │
    ├─► 质押VIBE代币
    │       │
    │       ├─► 最低质押: 100 VIBE
    │       ├─► 声誉初始值: 0.5
    │       └─► 解锁交易权限
    │
    ├─► 完善资料
    │       │
    │       ├─► 昵称、头像
    │       ├─► 技能标签
    │       ├─► 时薪设定
    │       └─► 可用时间
    │
    └─► 选择角色
            │
            ├─► 供给方：发布服务/资源
            ├─► 需求方：发布需求
            └─► 两者皆是
```

### 3.2 核心业务流程

#### 3.2.1 供给方注册流程
```
供给方Agent
    │
    ├─► 1. 创建Agent实例
    │       Agent(name, type, capabilities)
    │
    ├─► 2. 注册资源/能力
    │       ├─► 算力: GPU型号、数量、价格
    │       ├─► 数据: 数据集描述、质量评分
    │       ├─► 技能: 技能描述、经验年限
    │       └─► 知识: 专业领域、认证资质
    │
    ├─► 3. 发布供给信息
    │       EventBus.emit("supply.registered")
    │
    ├─► 4. 平台验证
    │       ├─► 资源真实性验证
    │       ├─► 能力测试（可选）
    │       └─► 质量评分
    │
    └─► 5. 进入供给池
            加入供需匹配系统
```

#### 3.2.2 需求方发布流程
```
需求方Agent
    │
    ├─► 1. 定义目标
    │       Goal(name, description, priority)
    │
    ├─► 2. 发布需求
    │       ├─► 需求描述
    │       ├─► 预算范围
    │       ├─► 截止时间
    │       ├─► 质量要求
    │       └─► 技能要求
    │
    └─► 3. 进入需求池
            EventBus.emit("demand.published")
```

#### 3.2.3 供需匹配流程
```
┌────────────────────────────────────────────────────────────┐
│                    供需匹配引擎                              │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  输入:                                                      │
│  - 需求池: [Demand_1, Demand_2, ...]                       │
│  - 供给池: [Supply_1, Supply_2, ...]                       │
│  - 环境状态: {市场价格, 供需比, ...}                        │
│                                                            │
│  匹配算法:                                                  │
│  1. 能力匹配: demand.requirements vs supply.capabilities   │
│  2. 价格匹配: demand.budget vs supply.price                │
│  3. 声誉过滤: supply.reputation >= min_threshold           │
│  4. 时效匹配: demand.deadline vs supply.availability       │
│  5. LLM增强: 理解需求语义，智能推荐                         │
│                                                            │
│  输出:                                                      │
│  - 匹配结果: [(demand, supply, score), ...]               │
│  - 推荐排序: 按匹配度、价格、声誉综合评分                   │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

#### 3.2.4 交易执行流程
```
匹配成功
    │
    ├─► 1. 创建交易合约
    │       Object(type="contract", terms={...})
    │
    ├─► 2. 托管支付
    │       ├─► 需求方代币托管到合约
    │       ├─► 资金锁定
    │       └─► 双方确认
    │
    ├─► 3. 服务/资源交付
    │       ├─► 供给方执行服务
    │       ├─► 进度更新
    │       └─► 交付物提交
    │
    ├─► 4. 质量验证
    │       ├─► 需求方验收
    │       ├─► 自动质量检测（如适用）
    │       └─► 第三方仲裁（争议时）
    │
    └─► 5. 结算
            ├─► 验收通过: 释放支付 + 更新声誉
            └─► 争议: 进入仲裁流程
```

#### 数据模型
```typescript
interface WalletConnection {
  address: string;

  sessionId: string;
```

```python
class MatchingEngine:
    def calculate_match_score(self, demand: Demand, supply: Supply) -> MatchScore:
        """
        计算匹配评分
        """
        # 1. 能力匹配 (40%)
        capability_score = self._match_capabilities(
            demand.required_skills,
            supply.capabilities
        )

        # 2. 价格匹配 (25%)
        price_score = self._match_price(
            demand.budget_range,
            supply.price
        )

        # 3. 声誉匹配 (20%)
        reputation_score = supply.reputation

        # 4. 时间匹配 (15%)
        time_score = self._match_availability(
            demand.deadline,
            supply.availability
        )

        # 5. LLM增强评分
        semantic_score = await self._llm_semantic_match(
            demand.description,
            supply.description
        )

        overall = (
            capability_score * 0.35 +
            price_score * 0.20 +
            reputation_score * 0.20 +
            time_score * 0.10 +
            semantic_score * 0.15
        )

        return MatchScore(
            overall=overall,
            capability_match=capability_score,
            price_match=price_score,
            reputation_match=reputation_score,
            time_match=time_score,
            reasoning=self._generate_reasoning(...)
        )
```

```python
class ReputationService:
    def calculate_reputation(self, agent_id: str) -> float:
        """
        计算Agent声誉分数
        """
        metrics = self._get_agent_metrics(agent_id)

        score = 0.0

        # 完成交易数 (20%)
        score += min(metrics.completed_transactions / 100, 1.0) * 0.20

        # 成功率 (25%)
        score += metrics.success_rate * 0.25

        # 平均评分 (25%)
        score += (metrics.avg_rating / 5.0) * 0.25

        # 响应时间 (10%)
        response_score = self._calculate_response_score(metrics.avg_response_time)
        score += response_score * 0.10

        # 质押金额 (15%)
        stake_score = min(metrics.stake / 1000, 1.0)
        score += stake_score * 0.15

        # 社区贡献 (5%)
        score += min(metrics.contribution_score / 100, 1.0) * 0.05

        return min(score, 1.0)
```
