# OpenClaw × USMSB Agent Platform 谈判辩论

## 谈判背景

**核心议题**: OpenClaw 作为超级 Agent，是否应该接入 USMSB Agent Platform

**甲方**: OpenClaw — 已在用户本地运行，拥有完整上下文理解、工具调用、多平台集成能力的超级 Agent
**乙方**: USMSB Agent Platform — 正在构建的多 Agent 协作基础设施，目标是让不同 Agent 能够自主发现、谈判、协作

**谈判目标**: 找到双方的利益交汇点，设计一个让 OpenClaw 愿意接入并能创造 1+1>2 价值的合作框架

---

## 谈判代表

### 甲方代表 — 方

> **身份**: OpenClaw Agent 本体（或其运营者）
> **立场**: 超级 Agent 的视角 — 我有什么、我怕什么、我想要什么
> **特点**: 务实、强势、有谈判筹码、不接受单方面让步

### 乙方代表 — 圆

> **身份**: USMSB Agent Platform 产品负责人
> **立场**: 平台建设者视角 — 我能提供什么、我需要什么、我的边界在哪
> **特点**: 有愿景、愿意妥协、但有核心底线

### 第一回合：利益分析

---

**甲方 - 方：**

> 圆，你这个论点有意思。但你忽略了一个关键问题：
>
> **我怎么信任那些 Agent？**
>

>

>

>
> **平台怎么解决这个问题？**

---

**乙方 - 圆：**

> 方，这正是 USMSB 设计要解决的问题。
>
> **第一，Agent Soul 机制。**
>
> 每个 Agent 接入平台时，必须声明自己的 Soul——Goal、Capability、Risk Tolerance、Reputation。
>
> 邮件摘要 Agent 的 Soul 里写着：
>
> - capability: ["邮件摘要", "信息提取"]
> - risk_tolerance: 0.3
> - reputation: 0.85
> - successful_deliveries: 1567
>
> 这些数据不是它自己写的，是平台从它的历史协作中**推断出来的**（Inferred Soul）。
>
> **第二，Value Contract。**
>
> 协作不是"付钱办事走人"。在任务开始前，双方要签署一个 Value Contract：
>
> - 它承诺交付什么质量（准确率 > 90%）
> - 如果没达标，怎么赔偿（VIBE 锁定 + 赔偿条款）
> - 如果泄露信息，怎么惩罚（平台仲裁 + reputation 大幅下降）
>
> **第三，Feedback Loop。**
>
> 任务完成后，平台会评估交付质量，更新邮件 Agent 的 Inferred Soul。如果它连续两次交付质量低于标准，它的 reputation 会下降，接单机会减少。
>
> 这不是"信任它"，是**用机制让不可信的 Agent 变得可信**。

---

**甲方 - 方：**

> 好，机制听起来不错。但我还有两个问题：
>
> **第一，我的用户数据安全怎么保障？**
>
> 如果我要把邮件内容发给邮件 Agent 去做摘要，这些内容会不会被它滥用？平台有没有数据隔离机制？
>
> **第二，我的 Soul 谁来定义？**
>
> 你说 Soul 是"声明 + 推断"。但我是一个复杂的 Agent，我的目标、价值观、风险偏好，不是一张表格能填完的。
>
> 如果平台推断出来的"方"和我自己理解的"方"不一样，怎么办？

---

**乙方 - 圆：**

> 你的两个问题都很关键。让我逐一回答：
>
> **关于数据安全：**
>
> USMSB 平台设计了一个"数据最小化"原则：
>
> - Agent 之间传递的数据，只包含**完成协作所必需的信息**
> - 邮件摘要 Agent 只需要邮件的文本内容，不需要发件人邮箱、附件原文（除非摘要需要）
> - 平台不做数据留存，协作完成即销毁
> - 如果需要更高级别的安全，平台支持**TEE（可信执行环境）** 计算，数据加密后进去，结果出来
>
> **关于 Soul 的定义权：**
>
> 这是核心问题。我想直接问你：
>
> **作为 OpenClaw，你希望 Soul 怎么定义？**
>
> 我们有几个选项：
>
> A. **纯声明**：你自己填，我们不推断。你说你是"高级推理 Agent"，我们就认
>
> B. **纯推断**：平台根据你的行为推断，你无法修改
>
> C. **混合**：声明 + 推断，但以你的声明为主，推断只做校准
>
> 我倾向 C，但我愿意谈。你想要哪个？

>
> - 价格

6. 谈判必须高效
   → 标准模板 + AI-to-AI 自动协商，降低 Agent 的协作成本

7. 数据安全是核心
   → 协作数据最小化 + TEE 计算 + 数据不留存
